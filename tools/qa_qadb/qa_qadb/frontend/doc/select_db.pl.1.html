SYNOPSIS

       select_db.pl - lists known QADB targets
       select_db.pl <n> - sets a QADB target

DESCRIPTION

       Multiple  scripts from the qa_tools package need to access QADB (mostly
       the script qa_db_report.pl(1) ).  This  script  allows  you  to  easily
       select  which  QADB  instance  to  use.   You can select the production
       database, or one of the test ones.

       Running the script  without  an  argument  lists  the  predefined  QADB
       instances.   When  the  predefined  instance is already selected, it is
       marked with an asterisk (*).

       Specifying a number <n> causes that you select the <n>-th database from
       the  list.   The database is then used to store your data, and to a few
       other minor purposes, like looking for known products.

DETAILS

       During the listing, the script just scans the file ~/.mysql_loc.rc  for
       one of the hardcoded predefined database instances, and marks the found
       one.

       When <n> was specified, the script rewrites  the  file  ~/.mysql_loc.rc
       with the <n>-th predefined hardcoded instance details.
